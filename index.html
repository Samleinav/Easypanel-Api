Introduction

This document provides an interactive guide to the unofficial Easypanel API. Since there is no official documentation, these endpoints have been determined by observing the web application's requests. The purpose of this application is to facilitate programmatic interaction with your Easypanel instance.

Base URLs

HTTP: http://<your-easypanel-ip>:3000/api/trpc
WebSocket: ws://<your-easypanel-ip>:3000/ws

Error Handling

The API uses standard HTTP status codes. Common errors include `400 Bad Request` for malformed requests and `401 Unauthorized` for authentication issues. The error response body will contain specific details.

400Bad Request

The request has invalid syntax or is missing parameters.

{
    "error": {
        "json": {
            "message": "[...]",
            "code": -32600,
            "data": { "code": "BAD_REQUEST", "httpStatus": 400, "path": "...", "zodErrors": {}}
        }
    }
}

401Unauthorized

The authentication token is missing, invalid, or has expired.

{
    "error": {
        "json": {
            "message": "UNAUTHORIZED",
            "code": -32001,
            "data": { "code": "UNAUTHORIZED", "httpStatus": 401, "path": "...", "zodErrors": null }
        }
    }
}

Login

Authenticates a user and returns a session token. This token must be included in the `Authorization` header for subsequent requests.

POST auth.login

Request Body

{
  "json": {
    "email": "admin@example.com",
    "password": "your-secure-password"
  }
}

Successful Response (200 OK)

{
    "result": {
        "data": {
            "json": {
                "token": "cmfszafj3001x07o8cy601e4y"
            }
        }
    }
}

Update App Source

Configures the source code or image for an application service. Use only one of the following methods.

From Docker Image

services.app.updateSourceImage 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "image": "nginx:latest",
    "username": "user (optional)",
    "password": "password (optional)"
  }
}

From Git Repository

services.app.updateSourceGit 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "repo": "https://github.com/user/repo.git",
    "ref": "main",
    "path": "/"
  }
}

From Dockerfile

services.app.updateSourceDockerfile 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "dockerfile": "FROM node:18\nCOPY . .\nRUN npm install\nCMD [\"npm\", \"start\"]"
  }
}

From Uploaded ZIP

This is a `multipart/form-data` request, not JSON.

/api/upload-code/{project-name}/{service-name}

Create from Schema

Creates one or more services within a project from a predefined schema. This is useful for deploying complex applications with multiple components, like n8n with a backing database.

POST templates.createFromSchema

Example: Deploying n8n

This example schema defines an n8n application service.

{
    "json": {
        "projectName": "my-n8n-project",
        "schema": {
            "services": [
                {
                    "type": "app",
                    "data": {
                        "projectName": "my-n8n-project",
                        "serviceName": "my-n8n-instance",
                        "source": {
                            "type": "image",
                            "image": "n8nio/n8n:latest"
                        },
                        "env": "DB_TYPE=postgresdb\\r\\nDB_POSTGRESDB_HOST=postgres.service.internal\\r\\n...",
                        "deploy": { "replicas": 1, "zeroDowntime": true },
                        "domains": [
                            {
                                "host": "n8n.$(EASYPANEL_DOMAIN)",
                                "https": true,
                                "port": 5678
                            }
                        ],
                        "resources": { "memoryReservation": 1024, "cpuReservation": 1 }
                    }
                }
            ]
        }
    }
}

Successful Response (200 OK)

{
    "result": {
        "data": {
            "json": {
                "type": "app",
                "serviceName": "my-n8n-instance"
            }
        }
    }
}

Note: Further exploration of schema details and advanced configurations is still pending.

Get Service Logs (WebSocket)

Streams real-time logs for a service. The service is identified by the format `{project_name}_{service_name}`.

WEBSOCKET /serviceLogs

Example Connection URL

ws://<ip>:3000/ws/serviceLogs?token={token}&service=my-project_my-app&compose=false
