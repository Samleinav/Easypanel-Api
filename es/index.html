Introducción

Este documento proporciona una guía interactiva para la API no oficial de Easypanel. Dado que no existe documentación oficial, estos endpoints se han determinado observando las solicitudes de la aplicación web. El propósito de esta aplicación es facilitar la interacción programática con su instancia de Easypanel.

URLs Base

HTTP: http://<your-easypanel-ip>:3000/api/trpc
WebSocket: ws://<your-easypanel-ip>:3000/ws

Manejo de Errores

La API utiliza códigos de estado HTTP estándar. Los errores comunes incluyen `400 Bad Request` para solicitudes malformadas y `401 Unauthorized` para problemas de autenticación. El cuerpo de la respuesta del error contendrá detalles específicos.

400Bad Request

La solicitud tiene una sintaxis no válida o faltan parámetros.

{
    "error": {
        "json": {
            "message": "[...]",
            "code": -32600,
            "data": { "code": "BAD_REQUEST", "httpStatus": 400, "path": "...", "zodErrors": {}}
        }
    }
}

401Unauthorized

El token de autenticación falta, es inválido o ha expirado.

{
    "error": {
        "json": {
            "message": "UNAUTHORIZED",
            "code": -32001,
            "data": { "code": "UNAUTHORIZED", "httpStatus": 401, "path": "...", "zodErrors": null }
        }
    }
}

Login

Autentica a un usuario y devuelve un token de sesión. Este token debe incluirse en el encabezado `Authorization` para las solicitudes posteriores.

POST auth.login

Cuerpo de la Solicitud

{
  "json": {
    "email": "admin@example.com",
    "password": "your-secure-password"
  }
}

Respuesta Exitosa (200 OK)

{
    "result": {
        "data": {
            "json": {
                "token": "cmfszafj3001x07o8cy601e4y"
            }
        }
    }
}

Actualizar Origen de la App

Configura el código fuente o la imagen para un servicio de aplicación. Utilice solo uno de los siguientes métodos.

Desde Imagen de Docker

services.app.updateSourceImage 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "image": "nginx:latest",
    "username": "user (opcional)",
    "password": "password (opcional)"
  }
}

Desde Repositorio Git

services.app.updateSourceGit 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "repo": "https://github.com/user/repo.git",
    "ref": "main",
    "path": "/"
  }
}

Desde Dockerfile

services.app.updateSourceDockerfile 
{
  "json": {
    "projectName": "my-project",
    "serviceName": "my-app-service",
    "dockerfile": "FROM node:18\nCOPY . .\nRUN npm install\nCMD [\"npm\", \"start\"]"
  }
}

Desde ZIP Subido

Esta es una solicitud `multipart/form-data`, no JSON.

/api/upload-code/{project-name}/{service-name}

Crear desde Schema

Crea uno o más servicios dentro de un proyecto a partir de un esquema predefinido. Esto es útil para desplegar aplicaciones complejas con múltiples componentes, como n8n con una base de datos de respaldo.

POST templates.createFromSchema

Ejemplo: Desplegando n8n

Este schema de ejemplo define un servicio de aplicación n8n.

{
    "json": {
        "projectName": "my-n8n-project",
        "schema": {
            "services": [
                {
                    "type": "app",
                    "data": {
                        "projectName": "my-n8n-project",
                        "serviceName": "my-n8n-instance",
                        "source": {
                            "type": "image",
                            "image": "n8nio/n8n:latest"
                        },
                        "env": "DB_TYPE=postgresdb\\r\\nDB_POSTGRESDB_HOST=postgres.service.internal\\r\\n...",
                        "deploy": { "replicas": 1, "zeroDowntime": true },
                        "domains": [
                            {
                                "host": "n8n.$(EASYPANEL_DOMAIN)",
                                "https": true,
                                "port": 5678
                            }
                        ],
                        "resources": { "memoryReservation": 1024, "cpuReservation": 1 }
                    }
                }
            ]
        }
    }
}

Respuesta Exitosa (200 OK)

{
    "result": {
        "data": {
            "json": {
                "type": "app",
                "serviceName": "my-n8n-instance"
            }
        }
    }
}

Nota: Aún está pendiente una exploración más detallada de los schemas y las configuraciones avanzadas.

Obtener Logs del Servicio (WebSocket)

Transmite logs en tiempo real para un servicio. El servicio se identifica con el formato `{project_name}_{service_name}`.

WEBSOCKET /serviceLogs

URL de Conexión de Ejemplo

ws://<ip>:3000/ws/serviceLogs?token={token}&service=my-project_my-app&compose=false
